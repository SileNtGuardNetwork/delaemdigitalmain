# 06_SECTION_FACTORY — Website Factory

## Purpose
Build every section after the hero as part of one conversion route, not as disconnected dark cards.

## Section Build Formula

For each screen:

1. Define the commercial job.
2. Define the user emotion or realization.
3. Lock the copy.
4. Choose the layout pattern.
5. Choose the visual motif.
6. Define the motion.
7. Define mobile behavior.
8. Implement in Next.js.
9. Run visual QA.

## Core Patterns

### Diagnostic Grid
Best for problem/loss sections.

### Route Map
Best for systems, methodology, client journey.

### Split Comparison
Best for before/after, old/new, chaos/control.

### Timeline
Best for process and delivery steps.

### Service Cards
Best for offer components and packages.

### Proof Cards
Best for cases, testimonials, metrics, artifacts.

### Founder Editorial
Best for authority, story, methodology.

### Accordion
Best for FAQ and objections.

### Intake Panel
Best for final CTA and lead form.

## Section Acceptance Criteria

A section is accepted only if:

- it has one clear job;
- it advances the visitor toward conversion;
- copy and visual structure support the same idea;
- mobile layout is planned;
- motion improves comprehension, not decoration;
- it does not repeat the same card grid pattern as the previous section.

## Reject Criteria

Reject if:

- section exists only for decoration;
- headline is generic;
- cards are visually repetitive;
- CTA is missing or unclear;
- mobile layout will become a long unreadable stack;
- visual pattern does not match the section job.

## Rhythm Rule
Alternate section energy:

1. Hero — strong promise.
2. Diagnostic — problem tension.
3. System — clarity.
4. Offer — concrete composition.
5. Comparison — contrast.
6. Process — confidence.
7. Packages — choice.
8. Proof — trust.
9. Founder — authority.
10. Audit — low-friction action.
11. Pricing — budget clarity.
12. FAQ — remove doubts.
13. Final CTA — conversion.

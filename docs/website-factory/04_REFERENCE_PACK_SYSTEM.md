# 04_REFERENCE_PACK_SYSTEM — Website Factory

## Purpose
Use references as precise inputs, not as a visual dump.

A reference pack guides composition, typography, mood, interaction, and niche signals. It does not replace strategy or copy.

## Rule
Use 3–5 references per screen. More than that usually creates visual noise.

## Reference Slots

### 1. Composition Reference
What to extract:
- grid;
- content balance;
- visual anchor;
- CTA placement;
- desktop/mobile layout logic.

### 2. Typography Reference
What to extract:
- headline scale;
- line-height;
- text contrast;
- hierarchy;
- spacing rhythm.

### 3. Visual Mood Reference
What to extract:
- atmosphere;
- light;
- palette;
- materials;
- depth;
- premium cues.

### 4. Interaction Reference
What to extract:
- reveal logic;
- hover states;
- scroll motion;
- transition softness;
- micro-interactions.

### 5. Niche Signal Reference
What to extract:
- industry-specific trust markers;
- visual language;
- proof format;
- object/asset usage.

## Reference Card Template

```md
Reference URL / Screenshot:
Used for:
What to extract:
What to avoid:
Target section:
Notes for Codex:
```

## Good Reference Usage
Good: “Use this for headline rhythm and left/right balance.”
Bad: “Make it like this site.”

Good: “Extract the diagnostic route pattern, not the colors.”
Bad: “Copy the whole block.”

## Anti-Patterns

Avoid:
- mixing unrelated styles;
- using too many references;
- copying brand identity;
- letting AI combine everything into a generic result;
- treating reference screenshots as final design.

## Output
A proper reference pack must produce clear implementation notes for Codex or Cursor.

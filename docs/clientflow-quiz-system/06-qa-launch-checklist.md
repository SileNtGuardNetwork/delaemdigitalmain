# QA and Launch Checklist

## Copy QA

- Clear first-screen promise
- No generic quiz language
- Questions are high-signal
- Contact gate explains value
- Result copy is useful and specific
- CTA follows from diagnosis
- Tone matches niche and brand

## Design QA

- Looks like a premium mini-product
- Does not feel like a template builder
- Mobile-first layout is clean
- First screen is strong
- Question screens are readable
- Result dashboard has clear hierarchy
- CTA is visible but not noisy
- Visual metaphor matches niche

## UX and performance QA

- No horizontal scroll
- Works on small mobile screens
- Buttons are thumb-friendly
- Loading states exist
- Back/forward behavior is controlled
- Submit does not double-send
- AI timeout has fallback
- Page is fast enough for paid traffic

## Technical QA

- Typecheck passes
- Build passes
- Server validation exists
- Config validation exists
- Consent is required before submit
- Anti-spam baseline works
- Duplicate guard works
- Errors are stable and user-safe

## Analytics QA

- quiz_viewed fires
- quiz_started fires
- question_answered fires
- contact_submitted fires
- quiz_completed fires
- lead_scored fires
- offer_routed fires
- result_viewed fires
- cta_clicked fires
- UTM is captured

## Integration QA

- Telegram lead card arrives
- CRM/webhook payload is correct
- Failed delivery is logged
- Retry path is planned
- Email/lead magnet delivery works when enabled

## Legal QA

- Privacy link exists
- Cookie link exists
- Consent text is visible
- Consent version is stored
- Cookie settings are available
- Marketing consent is separate when needed

## Acceptance threshold

Internal quality score must be at least 85/100 for client work and 90/100 for own case studies.

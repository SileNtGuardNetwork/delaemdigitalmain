# Production Pipeline

## Goal

Make every new niche quiz predictable, fast and premium instead of custom chaos.

## Pipeline

1. Client brief.
2. Niche research.
3. LazyWeb design research pass.
4. Copy Intelligence Pack.
5. Visual Intelligence Pack.
6. Quiz Config draft.
7. Scoring and routing model.
8. AI prompts and fallback results.
9. UI implementation.
10. Telegram and CRM integration.
11. Analytics setup.
12. Legal and consent setup.
13. QA pass.
14. Vercel preview.
15. Client review.
16. Production launch.
17. First report after traffic.
18. Optimization sprint.

## Required artifacts per quiz

- Art Direction Brief
- Niche Intelligence Pack
- Copy Pack
- Design Pack
- Quiz Config
- Result Screen Config
- Telegram Lead Card Template
- CRM Payload Spec
- Analytics Event Map
- Legal and Consent Config
- QA Report
- Launch Notes
- Performance Report

## LazyWeb usage

LazyWeb is used only during production, not runtime.

Purpose:

- research real UI patterns;
- avoid generic quiz-builder aesthetics;
- collect references for onboarding, diagnostics, forms, result dashboards and mobile UX;
- support design decisions with evidence.

## Standard production rule

No quiz goes to launch without passing QA and having a measurable reporting plan.

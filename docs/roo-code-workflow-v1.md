# Roo Code Workflow v1

## Стек
- VS Code
- Roo Code
- OpenRouter
- DeepSeek Chat V3.1

## Назначение связки
Дешёвый AI coding agent для проекта, предназначенный для локальной разработки без прямого вмешательства в продакшен-код.

## Правила безопасности
- Не работать напрямую в main
- Не трогать .env и secrets
- Не запускать destructive-команды
- Не делать commit/push/deploy без отдельного разрешения
- Изменения делать маленькими слоями

## Базовый workflow
1. **Audit** – анализ текущего состояния кода, выявление проблем и возможностей.
2. **Plan** – создание плана изменений, разбиение на мелкие задачи.
3. **Implementation** – выполнение изменений строго в рамках разрешённых областей.
4. **Checks** – запуск разрешённых проверок (типизация, сборка, статус Git).
5. **Review** – проверка результата перед следующим шагом.

## Разрешённые проверки
- `npx tsc --noEmit`
- `npm run build`
- `git status --short`

## Запрещённые действия без отдельного разрешения
- `git reset`
- `git clean`
- `git push`
- `rm -rf`
- `npm install`
- `vercel --prod`
- `supabase db reset`
- SQL‑команды `DROP` / `DELETE FROM`

## Краткий вывод
Эта связка не заменяет Codex полностью, но даёт дешёвый рабочий агентный слой для локальной разработки. Позволяет ускорить рутинные задачи, сохраняя контроль над критическими операциями.

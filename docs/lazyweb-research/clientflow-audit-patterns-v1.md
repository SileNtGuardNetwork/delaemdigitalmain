# ClientFlow / Audit Patterns Research v1

## Research Status

**Lazyweb findings: pending.**

Lazyweb skills are installed in this environment, but callable Lazyweb screenshot MCP tools were not exposed during this pass. No Lazyweb screenshots or database results were used. This document is a structured research brief for the next pass, not a claim of completed external visual research.

## Research Goal

Find visual patterns that can inform the D3.3 ClientFlow System and Audit upgrade without drifting into fake dashboards, generic SaaS cards, or metric decoration.

Target direction:

- controlled ClientFlow architecture;
- compact diagnostic surfaces;
- meaningful route visualization;
- premium dark B2B atmosphere;
- mobile compression without card towers.

## Search Targets

### Diagnostic Dashboard Patterns

Look for:

- readiness states;
- diagnostic summaries;
- issue grouping;
- compact status rows;
- non-finance system health views.

Avoid:

- fake analytics charts;
- trading/dashboard overload;
- metrics with no source;
- colorful SaaS widgets.

### System Readiness Patterns

Look for:

- setup readiness checklists;
- deployment readiness screens;
- status progressions;
- route/state summaries.

Useful companies/categories to search:

- Linear;
- Vercel;
- Stripe onboarding/setup;
- incident/status products;
- developer infrastructure tools.

### Funnel / Lead Route Visualization

Look for:

- source -> landing -> lead -> handoff flows;
- compact journey maps;
- pipeline diagrams that do not become CRM dashboards;
- route-line patterns with grouped stages.

Avoid:

- marketing funnel clip-art;
- huge arrows;
- decorative process charts.

### Onboarding Checklist Patterns

Look for:

- guided setup flows;
- compact task groups;
- progressive disclosure;
- clear next action.

Use for:

- Audit section outcomes;
- "what happens next" clarity;
- mobile compression.

### B2B Audit Report Patterns

Look for:

- executive diagnostic summaries;
- issue/prioritization tables;
- recommendation cards with controlled hierarchy;
- report-like surfaces.

Avoid:

- PDF-like dullness;
- too many equal cards;
- corporate light-theme generic design.

### Product Command Center Patterns

Look for:

- controlled dark operational layouts;
- minimal navigation/status rails;
- system maps;
- architectural grids.

Avoid:

- literal command-center widgets;
- fake terminal/cyber aesthetics;
- neon green or violet identity.

### Mobile Diagnostic Compression Patterns

Look for:

- horizontal compact status strips;
- single-column priority summaries;
- collapsible diagnostic groups;
- fewer stronger checkpoints.

Avoid:

- endless vertical card stacks;
- cramped labels;
- tiny unreadable technical UI.
